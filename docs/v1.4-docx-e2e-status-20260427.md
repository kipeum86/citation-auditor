## Vendor Decision

각 legal agent의 vendored copy는 지금 전면 업데이트하지 않는다.

Current decision:

- Global/plugin path만 쓰는 프로젝트는 본체 업데이트로 충분하다.
- 기존 markdown 감사가 안정적으로 동작 중인 legal agent는 v1.3 vendored copy를 유지한다.
- DOCX 직접 감사가 실제로 필요한 repo 1개만 canary로 업데이트한다.
- Canary can proceed when a repo actually needs DOCX auditing.

Canary 조건:

- 해당 repo에 실제 DOCX 감사 수요가 있다.
- `.claude/skills/citation-auditor/`에 repo별 local customization이 없거나, 덮어쓰기 전 diff를 확인했다.
- 먼저 dry run을 실행한다.

  ```bash
  ./scripts/vendor-into.sh --dry-run /path/to/canary-repo
  ```

- v1.3 vendored copy 위에 적용할 때는 명시적으로 확인 플래그를 사용한다.

  ```bash
  ./scripts/vendor-into.sh --confirm-docx-upgrade /path/to/canary-repo
  ```

- canary repo 안에서 markdown fixture와 DOCX fixture를 모두 실제 slash command로 검증한다.
- 며칠 사용 후 문제가 없으면, 나머지는 필요할 때만 순차 업데이트한다.
